# Описание класса: MainActivity

________________________________________
## 1. Общая информация
*   **Имя класса:** `MainActivity`
*   **Тип:** `Activity` (Активность)
*   **Назначение:** Это главный экран приложения. Он выполняет роль «центра управления», где пользователь взаимодействует с графиком, настраивает параметры расчета и видит результаты.
*   **Взаимодействие:** 
    *   Использует `GraphView` для отображения и рисования кривой.
    *   Использует `HistoryDbHelper` для сохранения результатов расчетов.
    *   Запускает `HistoryActivity` для просмотра истории.

________________________________________
## 2. Переменные (Поля класса)

| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `graphView` | `GraphView` | Пользовательский компонент для рисования кривой и прямоугольников. | `onCreate`, `calculateArea`, `clearButton` |
| `rectanglesSeekBar` | `SeekBar` | Ползунок для выбора количества прямоугольников. | `onCreate`, `calculateArea` |
| `methodRadioGroup` | `RadioGroup` | Группа кнопок для выбора метода (Левый, Правый, Средний). | `calculateArea` |
| `dbHelper` | `HistoryDbHelper` | Объект для работы с базой данных SQLite. | `onCreate`, `saveCalculationToDb`, `onDestroy` |
| `currentCurveId` | `String` | Уникальный идентификатор (UUID) текущей кривой. | `startNewCurve`, `saveCalculationToDb` |

________________________________________
## 3. Методы класса

### Имя метода: `onCreate`
*   **Тип:** `protected`
*   **Возвращаемое значение:** `void` (ничего)
*   **Параметры:** `Bundle savedInstanceState` (сохраненное состояние).
*   **Что делает:** Инициализирует интерфейс, настраивает обработчики нажатий кнопок и ползунка, создает помощника базы данных.
*   **Когда вызывается:** Автоматически системой при запуске приложения.

### Имя метода: `calculateArea`
*   **Тип:** `private`
*   **Возвращаемое значение:** `void`
*   **Что делает:** 
    1. Получает точки кривой из `graphView`.
    2. Определяет выбранный метод и количество прямоугольников.
    3. Делит ось X на равные отрезки.
    4. Для каждого отрезка находит высоту, используя метод `findYForX`.
    5. Рассчитывает площадь прямоугольника и добавляет её к общей сумме.
    6. Обновляет экран и сохраняет данные в базу.
*   **Когда вызывается:** При нажатии на кнопку «Calculate».

### Имя метода: `findYForX`
*   **Тип:** `private`
*   **Возвращаемое значение:** `float` (координата Y для заданного X).
*   **Параметры:** 
    * `points`: список точек кривой.
    * `x`: искомая координата X.
*   **Что делает:** Использует **линейную интерполяцию**. Так как пользователь рисует точки с пропусками, этот метод вычисляет точное значение Y на линии между двумя записанными точками.
*   **Важно:** Это математическое «сердце» точности расчетов.

________________________________________
## 4. Жизненный цикл
*   **`onCreate()`**: Вызывается при создании экрана. Здесь мы создаем «каркас» нашего приложения.
*   **`onDestroy()`**: Вызывается перед закрытием. Мы закрываем базу данных, чтобы не было утечек памяти.

________________________________________
## 5. Взаимодействие с интерфейсом (UI)
*   **Кнопки:** `calculateButton`, `clearButton`, `historyButton`.
*   **SeekBar:** Служит для установки точности (количества прямоугольников).
*   **ListView:** Показывает краткую историю расчетов на текущем экране.

________________________________________
## 6. Взаимодействие с другими компонентами
*   **Intent:** Используется для перехода на экран `HistoryActivity`.
*   **База данных:** Сохраняет каждый расчет в файл `History.db`.
*   **GSON:** Превращает список объектов-точек в текстовую строку JSON для хранения в БД.

________________________________________
## 7. Общая логика
1. Пользователь рисует линию пальцем.
2. Выбирает метод «Middle» и 10 прямоугольников.
3. Нажимает кнопку расчета.
4. Приложение вычисляет площадь и рисует зеленые прямоугольники поверх графика.
5. Результат сохраняется в историю.

________________________________________
## 8. Объяснение простыми словами
Представьте, что у вас есть огород кривой формы, и вы хотите узнать его площадь. Так как это не ровный квадрат, вы ставите внутри него несколько прямоугольных досок. Вы измеряете высоту огорода в центре каждой доски. Сложив площади всех досок, вы получите примерный размер огорода. `MainActivity` — это тот самый «садовник», который делает все замеры и вычисления.
