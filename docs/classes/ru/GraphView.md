# Описание класса: GraphView

________________________________________
## 1. Общая информация
*   **Имя класса:** `GraphView`
*   **Тип:** `View` (Пользовательский компонент)
*   **Назначение:** Этот класс отвечает за визуальную часть приложения. Он управляет логикой отрисовки: осей (X и Y), кривой, нарисованной пользователем, и рассчитанных прямоугольников. Также он «слушает» прикосновения пользователя, чтобы записывать линию.
*   **Взаимодействие:** 
    *   Управляется классами `MainActivity` и `CurveDetailsActivity`.
    *   Предоставляет координаты кривой в `MainActivity` для расчета площади.

________________________________________
## 2. Переменные (Поля класса)

| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `curvePaint` | `Paint` | Определяет стиль (цвет, толщина) для синей кривой. | `init`, `onDraw` |
| `rectanglePaint` | `Paint` | Определяет стиль для полупрозрачных зеленых прямоугольников. | `init`, `onDraw` |
| `curvePath` | `Path` | Геометрический путь, соединяющий все точки, нарисованные пользователем. | `init`, `onDraw`, `onTouchEvent` |
| `curvePoints` | `List<PointF>` | Список точных координат (X, Y) кривой. | `onTouchEvent`, `getCurvePoints` |
| `isEditable` | `boolean` | Флаг включения/выключения рисования (например, выключено при просмотре истории). | `onTouchEvent`, `setEditable` |

________________________________________
## 3. Методы класса

### Имя метода: `onDraw`
*   **Тип:** `protected`
*   **Возвращаемое значение:** `void`
*   **Параметры:** `Canvas canvas` («холст» для рисования).
*   **Что делает:** 
    1. Рисует черные оси с стрелками и подписями (X, Y).
    2. Проходит циклом по списку `rectangles` и рисует каждый с зеленой заливкой и серой рамкой.
    3. Рисует `curvePath` (синюю линию).
*   **Когда вызывается:** Автоматически системой каждый раз, когда нужно обновить экран (например, при движении пальца или после расчета).

### Имя метода: `onTouchEvent`
*   **Тип:** `public`
*   **Возвращаемое значение:** `boolean` (True, если событие обработано).
*   **Параметры:** `MotionEvent event` (содержит данные о том, где и как пользователь коснулся экрана).
*   **Что делает:** 
    *   `ACTION_DOWN`: При первом касании очищает старую кривую и начинает новую.
    *   `ACTION_MOVE`: При движении пальца добавляет линии в `curvePath` и записывает точки в `curvePoints`.
*   **Когда вызывается:** Каждый раз, когда пользователь касается области графика.

### Имя метода: `setCurve`
*   **Тип:** `public`
*   **Возвращаемое значение:** `void`
*   **Параметры:** `List<PointF> points`
*   **Что делает:** Восстанавливает `curvePath` из сохраненного списка точек. Используется при загрузке данных из истории.

________________________________________
## 7. Общая логика
1. Пользователь касается экрана.
2. `onTouchEvent` записывает точки и создает `Path`.
3. Вызывается `invalidate()`, что запускает `onDraw`.
4. `onDraw` отрисовывает всё на экране, чтобы пользователь видел свою линию.

________________________________________
## 8. Объяснение простыми словами
`GraphView` — это как **умная школьная доска**. 
*   Она умеет сама рисовать свои границы и оси.
*   Она следит за вашим мелом (пальцем) и точно запоминает, где вы провели линию.
*   Когда «математическая часть» приложения решает положить зеленые плитки (прямоугольники) на доску, `GraphView` отображает их именно там, где нужно.
*   Она также может «запираться», чтобы вы могли смотреть старые рисунки, не размазав их случайно.
