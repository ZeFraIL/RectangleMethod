# Описание класса: HistoryContract

________________________________________
## 1. Общая информация
*   **Имя класса:** `HistoryContract`
*   **Тип:** `Normal Class` (Метаданные / Контракт)
*   **Назначение:** Этот класс служит **Определением схемы**. Он задает имена таблиц и колонок в базе данных в виде констант.
*   **Взаимодействие:** Используется каждым классом, который касается базы данных.

________________________________________
## 3. Общая логика
Этот класс сам по себе ничего не «делает». Он нужен для того, чтобы, если мы захотим изменить имя колонки с `total_area` на `calculated_area`, нам пришлось бы менять это только в **одном месте**, и всё приложение обновилось бы автоматически. Это защищает от опечаток и ошибок.

________________________________________
## 8. Объяснение простыми словами
Представьте, что вы с друзьями строите секретную базу и вам нужны **кодовые имена** для всего. Вместо того чтобы пытаться их запомнить, вы записываете их все на один лист бумаги. Когда кому-то нужно упомянуть «Синюю папку», он смотрит на лист, чтобы убедиться, что использует правильное имя. `HistoryContract` — это и есть тот самый лист бумаги.
