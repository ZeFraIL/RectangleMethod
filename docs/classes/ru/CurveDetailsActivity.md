# Описание класса: CurveDetailsActivity

________________________________________
## 1. Общая информация
*   **Имя класса:** `CurveDetailsActivity`
*   **Тип:** `Activity`
*   **Назначение:** Этот экран показывает детали конкретной сохраненной кривой. Он заново рисует график и выводит список всех расчетов (разными методами или количеством прямоугольников), сделанных для этой кривой.
*   **Взаимодействие:** 
    *   Использует `GraphView` в режиме «только для чтения».
    *   Запрашивает из базы данных данные, связанные с конкретным `CURVE_ID`.

________________________________________
## 2. Переменные (Поля класса)

| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `curveGraphView` | `GraphView` | Отображает статичную (нередактируемую) кривую. | `onCreate`, `loadCurve` |
| `calculations` | `ArrayList<String>` | Список текстовых описаний расчетов (например, «Left, 10 rects: 150.5»). | `loadCalculations` |

________________________________________
## 3. Методы класса

### Имя метода: `loadCurve`
*   **Тип:** `private`
*   **Возвращаемое значение:** `void`
*   **Параметры:** `String curveId`
*   **Что делает:** 
    1. Находит первую запись в БД для этого ID.
    2. Извлекает JSON-строку с точками.
    3. Использует **GSON**, чтобы превратить JSON обратно в список объектов `PointF`.
    4. Передает список в `curveGraphView.setCurve()`.

### Имя метода: `loadCalculations`
*   **Тип:** `private`
*   **Возвращаемое значение:** `void`
*   **Что делает:** Извлекает из базы все строки, где совпадает `curveId`. Собирает название метода, количество прямоугольников и общую площадь для каждого расчета и выводит их в список на экране.

________________________________________
## 6. Взаимодействие с другими компонентами
*   **Intent:** Получает данные от `HistoryActivity`.
*   **GSON:** Используется для «десериализации» (превращения текста обратно в программные объекты).

________________________________________
## 7. Общая логика
1. Активность запускается и получает ID.
2. Она рисует старую кривую на экране.
3. Под графиком выводит список всех площадей, которые когда-либо считались для этой линии.

________________________________________
## 8. Объяснение простыми словами
Это как **Страница деталей** конкретного проекта. Представьте, что вы сохранили рисунок и несколько разных замеров для него. Этот экран достает рисунок из «папки» (базы данных) и показывает вам в точности то, что вы нарисовали, и все результаты вычислений, которые вы сохранили ранее.
