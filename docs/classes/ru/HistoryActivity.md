# Описание класса: HistoryActivity

________________________________________
## 1. Общая информация
*   **Имя класса:** `HistoryActivity`
*   **Тип:** `Activity`
*   **Назначение:** Этот экран отображает список всех уникальных идентификаторов (ID) кривых, сохраненных в базе данных. Он позволяет пользователю просматривать свои прошлые сессии расчетов.
*   **Взаимодействие:** 
    *   Читает данные из `HistoryDbHelper`.
    *   Запускает `CurveDetailsActivity` при нажатии на элемент списка.

________________________________________
## 2. Переменные (Поля класса)

| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `curvesListView` | `ListView` | Элемент интерфейса для отображения списка. | `onCreate` |
| `dbHelper` | `HistoryDbHelper` | Для подключения к базе данных. | `onCreate`, `loadCurveIds` |
| `curveIds` | `ArrayList<String>` | Список, хранящий уникальные ID из базы. | `loadCurveIds`, `OnItemClickListener` |
| `adapter` | `ArrayAdapter` | Посредник между списком `curveIds` и визуальным элементом `ListView`. | `onCreate`, `loadCurveIds` |

________________________________________
## 3. Методы класса

### Имя метода: `loadCurveIds`
*   **Тип:** `private`
*   **Возвращаемое значение:** `void`
*   **Что делает:** 
    1. Открывает базу данных для чтения.
    2. Выполняет запрос с параметром `distinct` (уникальность). Это важно, так как для одной кривой может быть несколько расчетов. Мы хотим видеть ID кривой в этом списке только один раз.
    3. Добавляет ID в список `curveIds`.
    4. Дает команду адаптеру обновить данные на экране.
*   **Когда вызывается:** В методе `onCreate` при запуске экрана.

________________________________________
## 6. Взаимодействие с другими компонентами
*   **Intent:** Передает выбранный `CURVE_ID` как дополнительный параметр («extra») в `CurveDetailsActivity`. Так следующий экран узнает, какую именно кривую нужно загрузить.

________________________________________
## 7. Общая логика
1. Пользователь открывает экран истории.
2. Приложение ищет в базе данных все уникальные ID.
3. Пользователь видит список кодов (UUID).
4. Нажимает на один из них, и открывается новый экран с деталями этой кривой.

________________________________________
## 8. Объяснение простыми словами
Представьте, что это **Оглавление** книги. Сама «книга» — это ваша база данных. Каждая запись в списке — это номер страницы или название главы. Здесь вы не видите всей истории, только заголовки. Нажав на заголовок, вы переходите на конкретную страницу, чтобы увидеть все подробности.
