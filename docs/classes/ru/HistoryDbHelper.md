# Описание класса: HistoryDbHelper

________________________________________
## 1. Общая информация
*   **Имя класса:** `HistoryDbHelper`
*   **Тип:** `SQLiteOpenHelper` (Помощник базы данных)
*   **Назначение:** Этот класс — «библиотекарь» приложения. Он отвечает за создание, обновление и управление файлом базы данных SQLite, в котором хранятся все расчеты.
*   **Взаимодействие:** 
    *   Используется `MainActivity` для сохранения данных.
    *   Используется `HistoryActivity` и `CurveDetailsActivity` для чтения данных.

________________________________________
## 3. Методы класса

### Имя метода: `onCreate`
*   **Тип:** `public`
*   **Возвращаемое значение:** `void`
*   **Параметры:** `SQLiteDatabase db` (объект базы данных).
*   **Что делает:** Выполняет SQL-команду для создания таблицы. Определяет колонки: `curve_id`, `curve_points`, `method` и `total_area`.
*   **Когда вызывается:** Автоматически при самом первом обращении приложения к базе данных.

________________________________________
## 8. Объяснение простыми словами
Представьте большой шкаф с документами в подвале приложения. `HistoryDbHelper` — это клерк, у которого есть ключи. При первой установке приложения клерк собирает этот шкаф. Когда кто-то хочет подшить новый отчет или найти старый, они обращаются к этому клерку. Он следит, чтобы каждый отчет попал в нужную папку и остался там в сохранности, даже если вы выключите свет (закроете приложение).
