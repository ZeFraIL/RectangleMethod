# תיאור מחלקה: HistoryActivity

________________________________________
## 1. מידע כללי
*   **שם המחלקה:** `HistoryActivity`
*   **סוג:** `Activity`
*   **ייעוד:** מסך זה מציג רשימה של כל מזהי העקומות (IDs) הייחודיים שנשמרו במסד הנתונים. הוא מאפשר למשתמשים לדפדף בין מפגשי חישוב קודמים.
*   **אינטראקציה:** 
    *   קוראת נתונים מ-`HistoryDbHelper`.
    *   מפעילה את `CurveDetailsActivity` כאשר לוחצים על פריט ברשימה.

________________________________________
## 2. משתנים (שדות המחלקה)

| שם | סוג | מטרה | איפה נעשה בו שימוש |
| :--- | :--- | :--- | :--- |
| `curvesListView` | `ListView` | אלמנט ה-UI שמציג את רשימת המזהים. | `onCreate` |
| `dbHelper` | `HistoryDbHelper` | להתחברות למסד הנתונים. | `onCreate`, `loadCurveIds` |
| `curveIds` | `ArrayList<String>` | הרשימה שמחזיקה את המזהים הייחודיים שנשלפו ממסד הנתונים. | `loadCurveIds`, `OnItemClickListener` |

________________________________________
## 3. שיטות המחלקה

### שם השיטה: `loadCurveIds`
*   **סוג:** `private`
*   **ערך חזרה:** `void`
*   **מה היא עושה:** 
    1. פותחת את מסד הנתונים במצב קריאה.
    2. מריצה **Distinct Query** (שאילתת ייחודיות). זה חשוב כי לעקומה אחת יכולים להיות מספר חישובים. אנחנו רוצים לראות את מזהה העקומה רק פעם אחת ברשימה זו.
    3. מוסיפה את המזהים לרשימת `curveIds`.
    4. מורה לאדפטר לרענן את המסך.
*   **מתי נקראת:** בתוך `onCreate` כשהפעילות מתחילה.

## 8. הסבר במילים פשוטות
חשבו על זה כאל **תוכן העניינים** של ספר. ה"ספר" הוא מסד הנתונים שלכם. כל כניסה ברשימה היא מספר עמוד או כותרת פרק. אתם לא רואים את כל הסיפור כאן, רק את הכותרות. כשאתם לוחצים על כותרת, אתם עוברים לעמוד הספציפי ההוא כדי לראות את כל הפרטים.
