## 🎯 Цель проекта
•	Приложение визуализирует процесс численного интегрирования методом прямоугольников. Оно позволяет пользователю вручную нарисовать произвольную кривую и рассчитать площадь под ней.

•	Эта задача важна для образовательных целей, помогая наглядно понять принципы математического анализа и приближенных вычислений.

•	Целевая аудитория: Студенты, преподаватели математики и все, кто интересуется вычислительной геометрией.
________________________________________
## 📌 Требования к приложению
**Функциональные требования**
•	Возможность рисования кривой пальцем на экране.
•	Выбор метода прямоугольников: Левый, Правый или Средний.
•	Настройка количества прямоугольников через SeekBar.
•	Расчет общей площади и визуализация прямоугольников.
•	Сохранение истории вычислений в базу данных SQLite.
•	Просмотр подробной информации о сохраненных кривых.

**Нефункциональные требования**
•	Производительность: Плавная отрисовка кривой в реальном времени.
•	Удобство использования: Интуитивно понятный интерфейс с минимальным количеством шагов для получения результата.
•	Надёжность: Сохранение данных при перезагрузке приложения.

## 🧠 Общая архитектура
•	**Выбранный подход:**  
MVC (Model-View-Controller)

•	**Почему выбран именно он:**  
Для проекта такого масштаба MVC обеспечивает четкое разделение между логикой отрисовки (View), данными (Model - SQLite/GSON) и логикой управления (Activity).

•	**Основные компоненты системы:**  
- `MainActivity`: Контроллер и основной интерфейс.
- `GraphView`: Кастомный UI компонент для графики.
- `HistoryDbHelper`: Слой работы с данными (SQLite).
- `GSON`: Для преобразования списков точек в JSON.

## 🧩 Подробное описание классов
### 📌 Класс: MainActivity
**Роль:**  
Главный экран приложения.

**Ответственность:**  
Обработка пользовательского ввода, управление процессом расчета, связь между UI и базой данных.

**Основные методы:**  
- `onCreate()` — Инициализация компонентов и слушателей.
- `calculateArea()` — Основная логика расчета площади методом прямоугольников с использованием интерполяции.
- `saveCalculationToDb()` — Сериализация точек кривой и сохранение результата.

**Взаимодействие с другими классами:**  
Получает данные о точках от `GraphView`, передает данные в `HistoryDbHelper` и открывает `HistoryActivity`.
________________________________________
### 📌 Класс: GraphView
**Роль:**  
Визуализатор графика.

**Почему используется:**  
Стандартные компоненты Android не поддерживают свободное рисование кривых с последующей математической обработкой координат.

**Основные методы:**  
- `onDraw()` — Отрисовка осей, кривой и прямоугольников.
- `onTouchEvent()` — Обработка жестов для рисования.
________________________________________
### 📌 Класс: HistoryDbHelper / Data Layer
**Роль:**  
Управление базой данных SQLite для хранения истории расчетов.
________________________________________
## 🔄 Диаграмма работы приложения
1. Пользователь рисует кривую на `GraphView`.
2. Пользователь выбирает метод и количество прямоугольников.
3. Нажимает "Calculate".
4. Система вычисляет высоту каждого прямоугольника через интерполяцию точек кривой.
5. Результат отображается на экране и сохраняется в БД.

## ⚙️ Работа с потоками
•	**Используются:**  
В текущей версии операции выполняются в основном потоке (UI Thread), так как расчеты для малого количества прямоугольников выполняются мгновенно.

•	**Как предотвращаются:**  
–	зависания (ANR): Использование оптимизированных циклов.
–	утечки памяти: Закрытие курсоров БД в `onDestroy()`.

•	Обеспечивается сохранность через стандартные механизмы SQLite.
